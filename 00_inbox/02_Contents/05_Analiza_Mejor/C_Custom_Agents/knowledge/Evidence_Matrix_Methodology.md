# Metodología Evidence Matrix: De datos caóticos a insights estructurados

## Propósito de la Evidence Matrix

Una Evidence Matrix es una herramienta de análisis que conecta **afirmaciones** (claims) con **evidencias** (evidence) y les asigna **niveles de confianza** (confidence levels). Convierte investigación caótica en conocimiento estructurado y auditable.

**Principio fundamental**: Si no puedes mostrar la evidencia que sustenta una afirmación, esa afirmación no pertenece a un informe profesional.

## Estructura de la Evidence Matrix

### Componentes básicos

```
┌──────────────────┬─────────────────────┬──────────────────┬────────────────┐
│ AFIRMACIÓN       │ EVIDENCIA           │ FUENTE           │ CONFIANZA      │
│ (Claim)          │ (Evidence)          │ (Source)         │ (Confidence)   │
├──────────────────┼─────────────────────┼──────────────────┼────────────────┤
│ [Lo que          │ [Dato/hecho         │ [De dónde viene] │ [Alta/Media/   │
│  afirmamos]      │  que lo sustenta]   │                  │  Baja]         │
└──────────────────┴─────────────────────┴──────────────────┴────────────────┘
```

### Niveles de confianza: Definición operativa

**ALTA** (🟢)
- Fuente primaria verificable (organismos oficiales, estudios peer-reviewed)
- Datos recientes (< 2 años)
- Metodología transparente
- Corroborado por múltiples fuentes independientes

**MEDIA** (🟡)
- Fuente secundaria confiable (consultoras reconocidas, medios especializados)
- Datos con 2-5 años de antigüedad
- Metodología no completamente transparente
- Corroborado por al menos una fuente adicional

**BAJA** (🔴)
- Fuente terciaria o indirecta
- Datos antiguos (> 5 años) o sin fecha
- Metodología opaca o inexistente
- Sin corroboración

## Cómo evaluar la calidad de la evidencia

### Criterios de evaluación (Framework CRAAP)

**C — Currency (Vigencia)**
- ¿Cuándo se publicó la información?
- ¿Ha sido actualizada?
- ¿Sigue siendo relevante?

**R — Relevance (Relevancia)**
- ¿La información responde directamente a la pregunta?
- ¿Es del nivel de profundidad adecuado?
- ¿El público objetivo de la fuente coincide con nuestras necesidades?

**A — Authority (Autoridad)**
- ¿Quién es el autor/organización?
- ¿Qué credenciales tiene?
- ¿Es reconocido en su campo?

**A — Accuracy (Precisión)**
- ¿La información está verificada?
- ¿Se citan fuentes?
- ¿Hay evidencia de revisión editorial o peer review?

**P — Purpose (Propósito)**
- ¿Por qué existe esta información?
- ¿Hay sesgo evidente (vender, persuadir, informar)?
- ¿Hay conflictos de interés?

### Jerarquía de fuentes (de mayor a menor confianza)

**Nivel 1: Fuentes primarias**
- Datos oficiales (INE, Eurostat, Banco Mundial)
- Estudios peer-reviewed
- Informes de organismos internacionales (ONU, OCDE, OMC)

**Nivel 2: Fuentes secundarias confiables**
- Informes de consultoras reconocidas (McKinsey, BCG, Deloitte)
- Medios especializados con reputación (The Economist, Financial Times)
- Asociaciones sectoriales establecidas

**Nivel 3: Fuentes secundarias moderadas**
- Noticias de medios generalistas
- Blogs de expertos reconocidos
- Informes de empresas privadas

**Nivel 4: Fuentes terciarias o cuestionables**
- Wikipedia (útil para empezar, no para citar)
- Redes sociales
- Sitios sin autor identificable
- Fuentes sin fecha de publicación

## Metodología: De caos a insight a implicación a decisión

### FASE 1: CAOS (Recopilación)

Después de la fase inicial de investigación tienes:
- 40 pestañas abiertas del navegador
- 15 PDFs descargados
- Notas desestructuradas
- Datos contradictorios

**Objetivo de esta fase**: Recopilar sin juzgar. Todo lo que parezca relevante.

### FASE 2: ESTRUCTURA (Evidence Matrix)

Convertir el caos en estructura:

1. **Listar todas las afirmaciones** que podrías hacer sobre tu tema
2. **Para cada afirmación, buscar la evidencia** que la sustenta
3. **Citar la fuente** de cada evidencia
4. **Asignar nivel de confianza** usando los criterios CRAAP

**Resultado**: Una matriz donde cada afirmación está sustentada (o descartada si no hay evidencia).

### FASE 3: INSIGHT (Síntesis)

Analizar la matriz para extraer insights:

**Preguntas clave:**
- ¿Qué patrones emergen?
- ¿Qué afirmaciones tienen mayor confianza?
- ¿Qué contradicciones hay?
- ¿Qué lagunas de información existen?

**Técnica del "So What?"** (¿Y qué?)
Para cada afirmación con confianza alta/media, preguntarse: "¿Y qué significa esto para mi objetivo?"

**Ejemplo:**
- Afirmación: "El mercado gaming en Alemania creció un 8% en 2025"
- So What?: "Hay demanda creciente → oportunidad para empresas canarias"
- So What? x2: "Si no actuamos ahora, otros países ocuparán el espacio"

### FASE 4: IMPLICACIÓN (Interpretación)

Convertir insights en implicaciones estratégicas:

**Estructura:**
```
INSIGHT: [Lo que descubrimos]
IMPLICACIÓN: [Lo que esto significa para PROEXCA/empresa canaria]
RECOMENDACIÓN: [Qué hacer al respecto]
```

**Ejemplo:**
```
INSIGHT: Alemania tiene incentivos fiscales para co-producciones audiovisuales
IMPLICACIÓN: Empresas canarias audiovisuales podrían acceder a financiación alemana
RECOMENDACIÓN: Organizar misión comercial con talleres sobre co-producción y fiscal
```

### FASE 5: DECISIÓN (Executive Memo)

La Evidence Matrix alimenta el Executive Memo final (ver metodología aparte).

## Checklist de calidad para Evidence Matrix

Antes de considerar completada una Evidence Matrix:

- [ ] Cada afirmación importante tiene al menos una evidencia
- [ ] Cada evidencia tiene fuente citada (con enlace/referencia completa)
- [ ] Cada claim tiene nivel de confianza asignado y justificado
- [ ] Las afirmaciones con confianza BAJA están marcadas para verificación
- [ ] No hay afirmaciones sin evidencia (si las hay, eliminarlas)
- [ ] Las contradicciones entre fuentes están documentadas
- [ ] La matriz está organizada por preguntas clave (no por fuentes)

## Errores comunes y cómo evitarlos

### Error 1: Confundir fuente con evidencia

❌ EVIDENCIA: "Según McKinsey..."
✅ EVIDENCIA: "El 65% de empresas FTSE 100 usan IA generativa (McKinsey, 2025)"

### Error 2: No diferenciar niveles de confianza

Todas las evidencias NO son iguales. Distinguir entre dato oficial y estimación de blog.

### Error 3: Organizar por fuentes en lugar de por preguntas

❌ Estructura: "Lo que dice McKinsey / Lo que dice Deloitte / Lo que dice..."
✅ Estructura: "Pregunta 1 / Pregunta 2 / Pregunta 3..."

### Error 4: No documentar las lagunas

Las lagunas de información SON información. Si no existe evidencia para algo importante, documentarlo: "[NO ENCONTRADO: datos de cuota de mercado por subsector]"

## Integración con otros frameworks

**Research Brief → Evidence Matrix → Executive Memo**

- El Research Brief define las preguntas clave
- La Evidence Matrix responde las preguntas con evidencias
- El Executive Memo sintetiza la matriz en recomendaciones accionables
